### 一、核心知识点与 408 考情剖析

我们按照你的分类，逐一击破。在复习时，请时刻在脑海中挂念三个维度：**时间复杂度、空间复杂度、稳定性**。

#### 1. 插入排序 (Insertion Sorts)

- **直接插入排序**
    
    - **知识点**：将未排序元素逐个插入到已排序序列中。
        
    - **易考点**：最好情况（正序）时间复杂度为 $O(n)$，比较次数 $n-1$，移动次数 0。
        
    - **易错点**：注意稳定性，它是**稳定**的。
        
- **折半插入排序**
    
    - **重难点**：利用二分查找寻找插入位置。
        
    - **易错点/408 常考**：折半插入**仅减少了比较次数**（降为 $O(n \log n)$），但**并没有减少移动次数**。总时间复杂度依然是 $O(n^2)$。它是**稳定**的。
        
- **希尔排序 (Shell Sort)**
    
    - **知识点**：缩小增量排序，本质是分组的直接插入排序。
        
    - **易考点**：408 极爱考**“给定增量序列，求某几趟排序后的数组状态”**。
        
    - **易错点**：它是**不稳定**的。时间复杂度无法准确得出，通常认为在 $O(n^{1.3})$ 左右。
        

#### 2. 交换排序 (Exchange Sorts)

- **冒泡排序 (Bubble Sort)**
    
    - **易考点**：每一趟能确定一个最大（或最小）元素到达最终位置。408 常考“如果一趟中没有发生任何交换，说明数组已经有序，可以提前结束”。
        
    - **稳定性**：**稳定**。
        
- **快速排序 (Quick Sort)**
    
    - **重难点（极其重要）**：Partition（划分）过程。408 最喜欢考手推快排的划分过程。
        
    - **易考点**：最好情况时间 $O(n \log n)$；**最坏情况（数组已有序或逆序）时间退化为 $O(n^2)$**。空间复杂度是递归栈的深度，最好 $O(\log n)$，最坏 $O(n)$。
        
    - **易错点**：它是**不稳定**的。快排是所有内部排序中平均性能最优的。
        

#### 3. 选择排序 (Selection Sorts)

- **简单选择排序**
    
    - **易考点**：**无论初始状态如何，比较次数始终是 $n(n-1)/2$**，时间复杂度严格为 $O(n^2)$。
        
    - **易错点**：它是**不稳定**的（例如序列 `[2, 2, 1]`，第一趟后变成 `[1, 2, 2]`，两个 2 的相对位置变了）。
        
- **堆排序 (Heap Sort)**
    
    - **重难点（极其重要）**：建堆过程（从最后一个非叶子节点开始向下调整 Sift Down）和排序过程（不断输出堆顶，将堆底元素移到堆顶再调整）。
        
    - **易考点**：建堆的时间复杂度是 $O(n)$，每次调整的时间复杂度是 $O(\log n)$，总时间 $O(n \log n)$。空间复杂度为 $O(1)$。
        
    - **易错点**：大根堆用于**递增**排序，小根堆用于**递减**排序。它是**不稳定**的。
        

#### 4. 归并排序 (Merge Sort)

- **知识点**：采用分治法，408 默认考察 2 路归并。
    
    - **重难点/易考点**：**空间复杂度为 $O(n)$**，这是所有常见排序中空间开销最大的。
        
    - **稳定性**：**稳定**。408 常结合外部排序的“归并树”进行考察。
        

#### 5. 基数排序 (Radix Sort) 与计数排序 (Counting Sort)

- _(注：你提到的“技术排序”应为“计数排序”)_
    
- **基数排序**
    
    - **知识点**：基于关键字各位的大小进行分配（Distribute）和收集（Collect）。
        
    - **易考点**：时间复杂度 $O(d(n+r))$，空间复杂度 $O(r)$，其中 $d$ 是趟数（位数），$r$ 是基数（例如十进制 $r=10$）。
        
    - **稳定性**：必须**稳定**，否则基于低位的排序结果会被高位打乱。
        
- **计数排序**
    
    - **知识点**：用一个辅助数组记录每个元素出现的次数。时间/空间复杂度均为 $O(n+k)$（$k$ 为数据范围）。不基于比较，**稳定**。

### 二、 408 考场实战：如何面对题目？

1. **秒杀“选型题”**：
    
    - 看到“内存极小 / 空间限制 $O(1)$” $\rightarrow$ 排除归并，优先考虑堆排序。
        
    - 看到“初始基本有序” $\rightarrow$ 优先选直接插入排序；最忌讳选快速排序（会退化）。
        
    - 看到“要求稳定” $\rightarrow$ 排除快排、堆排、希尔、简单选择。
        
    - 看到“找出前 $K$ 大的元素” $\rightarrow$ 堆排序（建大小为 $K$ 的小根堆）或快速排序的 Partition。
        
2. **手推“过程题”**：
    
    - 408 选择题常给一个序列，问“这是哪种排序第二趟的结果”。
        
    - **破题法**：
        
        - 前部（或后部）有序，且包含极值 $\rightarrow$ 冒泡或选择。
            
        - 有一个元素在其最终位置，将数组一分为二 $\rightarrow$ 快排。
            
        - 前几个元素有序，后半部分未动 $\rightarrow$ 插入排序。
            
        - 具有明显的跳跃特征 $\rightarrow$ 希尔排序。

### 三、核心 C++ 实现代码 (带详细注释)

在 408 的代码题中，通常要求手写清晰、健壮的 C 风格或 C++ 基础代码。以下是这些算法的经典实现，**注意：408 考试中数组通常习惯从下标 1 开始，这里为了符合现代 C++ 规范，全部采用 0 索引。**

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// ================== 1. 插入排序类 ==================

// 1.1 直接插入排序
void insertionSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 1; i < n; ++i) {
        int temp = arr[i]; // 待插入元素
        int j = i - 1;
        // 从后往前找插入位置，边找边往后挪
        while (j >= 0 && arr[j] > temp) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = temp;
    }
}

// 1.2 希尔排序
void shellSort(vector<int>& arr) {
    int n = arr.size();
    // 增量 d 每次减半
    for (int d = n / 2; d > 0; d /= 2) {
        // 对每个子序列进行直接插入排序
        for (int i = d; i < n; ++i) {
            int temp = arr[i];
            int j = i - d;
            while (j >= 0 && arr[j] > temp) {
                arr[j + d] = arr[j];
                j -= d;
            }
            arr[j + d] = temp;
        }
    }
}

// ================== 2. 交换排序类 ==================

// 2.1 冒泡排序 (带提前结束优化)
void bubbleSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; ++i) {
        bool swapped = false; // 标记本趟是否发生交换
        for (int j = 0; j < n - 1 - i; ++j) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break; // 若无交换，说明已有序，提前结束
    }
}

// 2.2 快速排序
// 划分函数 (Hoare 双指针法，408标准教材写法)
int partition(vector<int>& arr, int low, int high) {
    int pivot = arr[low]; // 通常取第一个元素作为枢轴
    while (low < high) {
        while (low < high && arr[high] >= pivot) high--; // 从右向左找比枢轴小的
        arr[low] = arr[high]; // 放到左边
        while (low < high && arr[low] <= pivot) low++;   // 从左向右找比枢轴大的
        arr[high] = arr[low]; // 放到右边
    }
    arr[low] = pivot; // 枢轴到位
    return low;       // 返回枢轴最终下标
}

void quickSort(vector<int>& arr, int low, int high) {
    if (low < high) {
        int pivotPos = partition(arr, low, high);
        quickSort(arr, low, pivotPos - 1); // 递归排序左子表
        quickSort(arr, pivotPos + 1, high);// 递归排序右子表
    }
}

// ================== 3. 选择排序类 ==================

// 3.1 简单选择排序
void selectionSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; ++i) {
        int minIdx = i;
        // 在无序区中找最小元素
        for (int j = i + 1; j < n; ++j) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        if (minIdx != i) swap(arr[i], arr[minIdx]);
    }
}

// 3.2 堆排序
// 向下调整函数 (Sift Down) - 维护大根堆
void heapify(vector<int>& arr, int n, int i) {
    int largest = i;       // 初始化 largest 为当前节点
    int left = 2 * i + 1;  // 左孩子 (0索引)
    int right = 2 * i + 2; // 右孩子 (0索引)

    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        swap(arr[i], arr[largest]);
        heapify(arr, n, largest); // 递归调整被破坏的子树
    }
}

void heapSort(vector<int>& arr) {
    int n = arr.size();
    // 1. 建堆：从最后一个非叶子节点开始向下调整
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
    // 2. 排序：将堆顶(最大值)交换到堆底，再调整剩下的堆
    for (int i = n - 1; i > 0; i--) {
        swap(arr[0], arr[i]);
        heapify(arr, i, 0); // 注意此时堆的大小变为 i
    }
}

// ================== 4. 归并排序 ==================

// 辅助数组，放在全局或传递均可，这里为了封装内部动态分配
void merge(vector<int>& arr, int low, int mid, int high, vector<int>& temp) {
    int i = low, j = mid + 1, k = low;
    // 将 arr 中的数据暂存到 temp 中
    for (int p = low; p <= high; p++) temp[p] = arr[p];
    
    // 双指针合并
    while (i <= mid && j <= high) {
        if (temp[i] <= temp[j]) arr[k++] = temp[i++]; // 加等号保证稳定性
        else arr[k++] = temp[j++];
    }
    while (i <= mid) arr[k++] = temp[i++];
    while (j <= high) arr[k++] = temp[j++];
}

void mergeSort(vector<int>& arr, int low, int high, vector<int>& temp) {
    if (low < high) {
        int mid = low + (high - low) / 2;
        mergeSort(arr, low, mid, temp);
        mergeSort(arr, mid + 1, high, temp);
        merge(arr, low, mid, high, temp);
    }
}
```

建议你在复习时，将快速排序的 `partition` 和堆排序的 `heapify` 默写几遍，这是大题最常借用的代码骨架。

### 四、汇总表
| **排序算法**   | **时间复杂度 (最好)** | **时间复杂度 (最坏)** | **时间复杂度 (平均)** | **辅助空间复杂度**          | **稳定性** |
| ---------- | -------------- | -------------- | -------------- | -------------------- | ------- |
| **直接插入排序** | $O(n)$         | $O(n^2)$       | $O(n^2)$       | $O(1)$               | 稳定      |
| **折半插入排序** | $O(n)$         | $O(n^2)$       | $O(n^2)$       | $O(1)$               | 稳定      |
| **希尔排序**   | -              | -              | 约 $O(n^{1.3})$ | $O(1)$               | **不稳定** |
| **冒泡排序**   | $O(n)$         | $O(n^2)$       | $O(n^2)$       | $O(1)$               | 稳定      |
| **快速排序**   | $O(n \log n)$  | $O(n^2)$       | $O(n \log n)$  | $O(\log n)$ ~ $O(n)$ | **不稳定** |
| **简单选择排序** | $O(n^2)$       | $O(n^2)$       | $O(n^2)$       | $O(1)$               | **不稳定** |
