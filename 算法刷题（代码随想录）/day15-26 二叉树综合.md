## [平衡二叉树](https://leetcode.cn/problems/balanced-binary-tree/)

一般的拍脑袋思路是，遍历二叉树，然后对每一个节点计算左右的最大高度。

但是计算一棵二叉树的最大深度也需要递归遍历这棵树的所有节点，如果对每个节点都算一遍最大深度，时间复杂度是比较高的。

所以最好的解法是反过来思考，只计算一次最大深度，计算的过程中在后序遍历位置顺便判断二叉树是否平衡：

对于每个节点，先算出来左右子树的最大高度，然后在后序遍历的位置根据左右子树的最大高度判断平衡性。

```cpp
class Solution {
public:
    bool isBalanced(TreeNode* root) {
        checkBalanced(root);
        return balanced;
    }

private:
    // 记录二叉树是否平衡
    bool balanced = true;

    // 输入一个节点，返回以该节点为根的二叉树的最大深度
    int checkBalanced(TreeNode* root) {
        if (root == nullptr) {
            return 0;
        }
        
        // if (!balanced) {
        // 随便返回一个值即可，旨在结束递归
        //     return -666;
        // }

        int leftMaxDepth = checkBalanced(root->left);
        int rightMaxDepth = checkBalanced(root->right);

        // 后序遍历位置
        // 如果左右最大深度大于 1，就不是平衡二叉树
        if (abs(rightMaxDepth - leftMaxDepth) > 1) {
            balanced = false;
        }

        return 1 + max(leftMaxDepth, rightMaxDepth);
    }
};
```

## [二叉树的所有路径](https://leetcode.cn/problems/binary-tree-paths/)

```cpp
class Solution {
    vector<string> res;
    vector<int> path; // 建议存 int，最后转 string 效率更高

    void traverse(TreeNode* root) {
        if (!root) return;

        // 1. 进入节点：做选择（入栈）
        path.push_back(root->val);

        // 2. 判定：是否是叶子
        if (!root->left && !root->right) {
            res.push_back(formatPath(path));
            // 注意：这里不需要 return，因为后面还有统一的出栈逻辑
        } else {
            // 3. 递归：向深处走
            traverse(root->left);
            traverse(root->right);
        }

        // 4. 离开节点：撤销选择（出栈）
        path.pop_back();
    }

    string formatPath(const vector<int>& p) {
        if (p.empty()) return "";
        stringstream ss;
        for (int i = 0; i < p.size(); ++i) {
            ss << p[i] << (i == p.size() - 1 ? "" : "->");
        }
        return ss.str();
    }

public:
    vector<string> binaryTreePaths(TreeNode* root) {
        res.clear();
        path.clear();
        traverse(root);
        return res;
    }
};
```

“做选择 -> 递归 -> 撤销选择”的模式是解决 **所有回溯类题目**的模板：

```cpp
void traverse(TreeNode* root) {
    if (!root) return;

    // 1. 【进】把当前节点放入路径
    path.push_back(root->val);

    // 2. 【查】如果是叶子，处理结果
    if (!root->left && !root->right) {
        res.push_back(format(path));
    } else {
        // 3. 【走】向深处探索
        traverse(root->left);
        traverse(root->right);
    }

    // 4. 【出】回溯！撤销刚才的选择（重要！）
    path.pop_back();
}
```

## [左叶子之和](https://leetcode.cn/problems/sum-of-left-leaves/)

```cpp
class Solution {
public:
    int sumOfLeftLeaves(TreeNode* root) {
        traverse(root);
        return sum;
    }

    // 记录左叶子之和
    int sum = 0;
    // 二叉树遍历函数
    void traverse(TreeNode* root) {
        if (root == nullptr) {
            return;
        }

        if (root->left != nullptr &&
            root->left->left == nullptr && root->left->right == nullptr) {
            // 找到左侧的叶子节点，记录累加值
            sum += root->left->val;
        }

        // 递归框架
        traverse(root->left);
        traverse(root->right);
    }
};
```

还有一种“统一逻辑”的写法，核心在于：**不让父节点去偷看子节点，而是让子节点带着“身份标签”进场。**

在之前的写法中，父节点必须判断 `root->left` 是否存在，再判断它是不是叶子。这其实打破了递归的独立性。

### 1. 统一递归版：带状态下行

我们给递归函数增加一个 `bool isLeft` 参数。这样，每个节点只需要关心自己：

1. 我是一个叶子吗？
2. 我是从左边来的吗？

```cpp
class Solution {
    int sum = 0;

    void traverse(TreeNode* root, bool isLeft) {
        if (!root) return;

        // 节点只检查自己：我是叶子且我是左孩子吗？
        if (!root->left && !root->right && isLeft) {
            sum += root->val;
            return; // 已经是叶子了，没必要往下递归
        }

        // 往下走时，明确告诉左孩子：你是左边来的；告诉右孩子：你是右边来的
        traverse(root->left, true);
        traverse(root->right, false);
    }

public:
    int sumOfLeftLeaves(TreeNode* root) {
        sum = 0;
        // 根节点虽然没有父节点，但它肯定不是“左叶子”，所以传 false
        traverse(root, false);
        return sum;
    }
};
```

### 2. 这种写法好在哪里？

- **逻辑高度解耦**：父节点不再需要关心子节点的内部结构（是否有左右孩子），它只负责传达“身份”。
- **代码极简**：你会发现 `if` 语句变得非常短，不再有一长串的 `root->left->left` 这种容易看花眼的指针操作。
- **易于扩展**：如果题目改成“求所有右叶子的和”，你只需要把 `isLeft` 判断改一下，逻辑完全通用。

### 3. 归纳：什么时候该用这种“带状态”的 DFS？

当你发现你的递归逻辑里出现了类似 `root->left->left` 这种“**跨级访问**”时，通常就是一个信号：**你应该尝试把信息作为参数传下去。**

常见场景包括：

- **257. 二叉树的所有路径**：传的是 `string path`（路径状态）。
- **98. 验证二叉搜索树**：传的是 `long min, long max`（数值边界状态）。
- **112. 路径总和**：传的是 `targetSum - root->val`（剩余数值状态）。

## [完全二叉树的节点个数](https://leetcode.cn/problems/count-complete-tree-nodes/)

使用框架：注意前序位置是进入节点的时候，所以这里我们乘以了 10, 而后序位置是离开这个节点，向上走（回溯），所以是离开节点，我们要除以 10。

![[Pasted image 20260210220002.png]]

```cpp
class Solution {
    int res = 0;      // 记录最终所有路径的总和
    int track = 0;    // 记录从根节点到当前节点的路径数字（状态）

public:
    int sumNumbers(TreeNode* root) {
        traverse(root);
        return res;
    }

    void traverse(TreeNode* root) {
        if (root == nullptr) {
            return;
        }

        // --- 前序位置 ---
        // 1. 进入节点：更新当前路径代表的数字
        track = track * 10 + root->val;

        // 2. 如果是叶子节点，把当前的路径数字累加到结果中
        if (root->left == nullptr && root->right == nullptr) {
            res += track;
        }

        traverse(root->left);
        // --- 中序位置 ---
        traverse(root->right);

        // --- 后序位置 ---
        // 3. 离开节点：回溯，撤销当前节点对数字的影响
        // 逻辑：123 变回 12，就是 (123 - 3) / 10
        track = (track - root->val) / 10;
    }
};
```

更优雅的写法：不需要全局变量，直接通过递归返回值汇总结果

```cpp
class Solution {
public:
    int sumNumbers(TreeNode* root) {
        return dfs(root, 0);
    }

    // prevSum 是父节点传下来的“累计数字”
    int dfs(TreeNode* root, int prevSum) {
        if (!root) return 0;

        // 1. 计算当前路径代表的数字
        int currentSum = prevSum * 10 + root->val;

        // 2. 如果是叶子节点，这就是一条完整路径的终点，返回这个数字
        if (!root->left && !root->right) {
            return currentSum;
        }

        // 3. 如果不是叶子，递归左右子树，并将结果相加
        return dfs(root->left, currentSum) + dfs(root->right, currentSum);
    }
};
```
